# Sakshi harness — единая система над семьёй проектов

## Что такое harness

Проекты — независимый флот: каждый живёт в своём git-репозитории, со своим CLAUDE.md и своим `.ai-factory/`. По-настоящему их не связывает ничто, кроме одной системы авторства — skills и orchestrator. Эта система и есть harness: она держит обвязку каждого проекта — его дерево CLAUDE.md и `.ai-factory/`.

## Wiring contract — правильное состояние координационного слоя

Правильное, единообразное состояние координационного слоя не привязано к конкретному проекту:

- **CLAUDE.md** — всегда загруженный ствол: harness подхватывает родительские CLAUDE.md вверх по дереву каталогов — то, что вложенные проекты лежат в отдельных git-репозиториях, загрузке не мешает (модель — [context-tree](../philosophy/context-tree.md)); один дом факта о проекте; описание поведения настоящим временем, открывающееся сразу содержанием — без бессодержательного заголовка или боилерплейт-интро.
- **AGENTS.md** — симлинк на соседний CLAUDE.md.
- **RULES.md** (`.ai-factory/RULES.md`) — обязательный к прочтению императивный канал orchestrator, читаемый из самого целевого репозитория (он НЕ parent-loaded); несёт только контр-дефолты (правило, которое исполнитель нарушил бы по своему дефолту И которое код сам по себе не научит, каждое — со своим «почему»); один дом факта — он никогда не повторяет то, что уже стоит в CLAUDE.md или ARCHITECTURE.md.
- **ARCHITECTURE.md** — границы модулей + выбранный паттерн; дом архитектуры.
- **Раскладка grove** — отдельные репозитории под координирующим корнем, root README § Setup гарантирует структуру чекаута + переключение на самую свежую ветку, так что ствол механически достигает каждой leaf-сессии (модель — [context-grove](../philosophy/context-grove.md)).
- Один дом факта — по всему слою целиком.

## Wiring map — что загружается и кто чем владеет

CLAUDE.md загружается в каждую сессию автоматически — ни один скилл его не доставляет, ребро — вложенность директорий. RULES.md и ARCHITECTURE.md читает orchestrator на Step 0 каждого из своих агентов (план / имплементация / ревью / тест) как обязательный контекст. Тег `Spec:` → task-spec → заметки, на которые он ссылается, образуют пройденную по ссылкам сеть. Роадмап читается по пути.

Владение, по указателю: `aif` скаффолдит координационный слой; orchestrator исполняет задачи роадмапа, читая каналы выше; цепочка планирования авторит артефакты (→ [skill-cycle](skill-cycle.md)); форма графа скилов (→ [skill-graph](skill-graph.md)).

## Возможности

Содержание проектных CLAUDE.md — за каждым проектом; harness им не распоряжается, его дело — обвязка. Одна система авторства над многими проектами даёт: провести конвенцию по всему флоту разом; обнаружить и починить дрейф; дедуплицировать по правилу «один дом факта»; сверить, что дерево проекта отвечает контракту по его текущему состоянию; холодно восстановить контекст любой leaf из одного её дерева.
