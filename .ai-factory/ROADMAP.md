## Test-coverage testability axis вЂ” friction, not DI-presence

## The paired loop вЂ” the relay marker kills classification drift

The architect keeps drifting off the relay discipline no matter how the prose is sharpened ([handoff 22](handoffs/22-architect-violated-relay-discipline-contaminated-editor.md)) вЂ” it authors briefs, leaks its inventory and collision-hints, re-sends skills a warm editor already has. Root cause is not weak wording but that the architect must **classify** every user message (editor-bound relay, or conversation aimed at me?) before relaying; more prose banning authored briefs pulls the same failing lever. This direction removes the judgment: a mechanical `::` marker decides what is relayed, so the architect never classifies. Hard rule: adding discrimination to the editor is rejected вЂ” it would compound drift from both sides.

### Phase 15 вЂ” `::` marks the relay; the architect stops guessing

Exactly two channels to the editor, no third: the **`::` relay** вЂ” a message ending with `::` is relayed, anything else is conversation, never forwarded вЂ” and the **authored apply work-order** (post-greenlight, pinned вЂ” unchanged). The trailing marker replaces the standing judgment handoff 22 shows the architect failing to honor; the full design is in the spec.

- [x] **15.1 вЂ” agent-architect: relay by `::` marker, not by classification** вЂ” the architect drifts ([handoff 22](handoffs/22-architect-violated-relay-discipline-contaminated-editor.md)) because it must *decide* which user messages are editor-bound relays before relaying them вЂ” that classification is the drift surface. Replace it with a marker: a user message ending with `::` is a relay вЂ” the whole message before the trailing marker is the payload (strip the marker, translate, expand a slash-command by-reference, add nothing); no trailing `::` в†’ conversation to the architect, never forwarded. The marker trails, never leads: a payload often opens with a slash-command the harness invokes only at message head вЂ” a leading `::` would strand the architect without the invoked skill. Exactly two editor channels remain вЂ” the `::` relay and the authored apply work-order. In `agent-architect/SKILL.md`: rewrite В§"Relay by default", delete the standing-judgment paragraph (65вЂ“74), drop В§"Spawn once"'s digest re-brief (a third channel); `editor.md` and `description:` unchanged. Guard: trailing `::` unconditional. Spec: `.ai-factory/specs/70-architect-relay-marker.md`. [28m 49s]
