> Generic AI Factory skills вЂ” reusable slash-command packages for Claude Code.

## Pyramid rewrite вЂ” the skill package on the written philosophy

The philosophy is written вЂ” [Skill pyramid](../docs/skill-pyramid.md), [Context tree](../docs/context-tree.md), grounded in the [Skill composition model](../docs/skill-composition-model.md) вЂ” but every skill and command in `src/` predates it. This direction rewrites the whole package to the pyramid. Hard rules for the direction: rewrites are behavior-first вЂ” a rewritten skill is unverified until a live run compares its output to the pre-rewrite baseline; mechanism never inlines into a top and a top never absorbs an engine's mass; `loads:` declarations and the engines' reverse-graph markers remain the dependency map throughout; the repo docs stay the philosophy's single home вЂ” rewrites are audited against them and never restate them; a link into a skill body only where it is load-bearing.

Opens with its own substrate (1.1вЂ“1.3) вЂ” the philosophy substrate lands before the rewrites read it; each rewrite then passes its live baseline before the next starts. The problem today: the package grew before the philosophy was named, and the pyramid inverts in places вЂ” `command-handoff` (140 lines) outweighs the `note` engine (115) it delegates to; top-level skills still carry mechanism of their own; the newest layers (folder-carried style, the walkable tree) are encoded only in the newest files.
