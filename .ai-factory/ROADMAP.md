## Test-coverage testability axis ‚Äî friction, not DI-presence

`roadmap-test-coverage` is stack-parametric everywhere except Layer 5, whose testability-review prompt is an un-abstracted slab of its NestJS origin ‚Äî a binary DI-presence axis phrased in TS idiom (`process.env`, `(req as any).user`, constructor-vs-module-singleton). A live orchestrator run this session fired `needs-refactor` off it on a Python milestone; the value "inject, don't read from a module singleton" maps badly onto Python, where a module singleton / `os.environ` / `datetime.now()` is trivially `monkeypatch`-able. The misfire still surfaced a real finding (a class self-acquiring its config), so the fix must keep that reachable without the false "untestable" framing. Hard rule for the direction: the silent-failure axis (`test-philosophy`, Layer 3) is a separate concern and stays untouched ‚Äî this is the orthogonal test-authoring-cost axis; and the verdict token grammar Layer 6/8 consume is a contract.

## The paired loop ‚Äî the relay marker kills classification drift

The architect keeps drifting off the relay discipline no matter how the prose is sharpened ([handoff 22](handoffs/22-architect-violated-relay-discipline-contaminated-editor.md)) ‚Äî it authors briefs, leaks its inventory and collision-hints, re-sends skills a warm editor already has. Root cause is not weak wording but that the architect must **classify** every user message (editor-bound relay, or conversation aimed at me?) before relaying; more prose banning authored briefs pulls the same failing lever. This direction removes the judgment: a mechanical `::` marker decides what is relayed, so the architect never classifies. Hard rule: adding discrimination to the editor is rejected ‚Äî it would compound drift from both sides.

### Phase 15 ‚Äî `::` marks the relay; the architect stops guessing

Exactly two channels to the editor, no third: the **`::` relay** ‚Äî a message ending with `::` is relayed, anything else is conversation, never forwarded ‚Äî and the **authored apply work-order** (post-greenlight, pinned ‚Äî unchanged). The trailing marker replaces the standing judgment handoff 22 shows the architect failing to honor; the full design is in the spec.

- [ ] **15.1 ‚Äî agent-architect: relay by `::` marker, not by classification** ‚Äî the architect drifts ([handoff 22](handoffs/22-architect-violated-relay-discipline-contaminated-editor.md)) because it must *decide* which user messages are editor-bound relays before relaying them ‚Äî that classification is the drift surface. Replace it with a marker: a user message ending with `::` is a relay ‚Äî the whole message before the trailing marker is the payload (strip the marker, translate, expand a slash-command by-reference, add nothing); no trailing `::` ‚Üí conversation to the architect, never forwarded. The marker trails, never leads: a payload often opens with a slash-command the harness invokes only at message head ‚Äî a leading `::` would strand the architect without the invoked skill. Exactly two editor channels remain ‚Äî the `::` relay and the authored apply work-order. In `agent-architect/SKILL.md`: rewrite ¬ß"Relay by default", delete the standing-judgment paragraph (65‚Äì74), drop ¬ß"Spawn once"'s digest re-brief (a third channel); `editor.md` and `description:` unchanged. Guard: trailing `::` unconditional. Spec: `.ai-factory/specs/70-architect-relay-marker.md`.
