## Milestones

- [x] **milestone-rescue: update JSON sidecar after artifact cleanup** — After deleting plan-review or review files, the orchestrator's `plans/{slug}.json` sidecar retains a stale `step` value referencing the deleted files. Fix: at the end of Step 5 (Apply and clean up), read the sidecar, determine the correct `step` from what remains on disk (plan-reviews deleted → `"planned"`; reviews deleted, plan-reviews pass → `"plan_reviewed"`), write only the `step` key back — keep `planner`/`implementer`/`elapsed` untouched. Spec: `.ai-factory/notes/08-milestone-rescue-sidecar-update.md`. [17m 24s]

- [x] **roadmap-decompose: enforce atomicity via deployability test** — `roadmap-decompose/SKILL.md` currently states "one atomic task / one concern / one reason to revert" as a declaration but has no enforcement step. The rule is too abstract to apply consistently — concerns slip in through "while I'm here" additions (e.g. adding a service method inside a component task). Fix: add a mandatory **Atomicity Gate** step that runs after each task description is drafted, before it is saved. The gate asks one question: **"Can the first half be deployed without the second half and still make sense?"** — if yes, split into two tasks. This is the only atomicity criterion; drop the file-count and verb-count heuristics as too noisy. Concretely: in Mode 1 (Step 1.3) after drafting each milestone, and in Mode 2 (Step 2.4) after drafting each new task — pause and apply the gate. If the answer is yes, split and apply the gate again to each half recursively until no half is independently deployable. Add this as **Step 1.3.1** and **Step 2.4.1** in the skill body. The gate question and split instruction should appear verbatim in the skill so the agent always uses the same wording.

- [x] **command-handoff: self-handoff prompt command that survives /compact** — There is no first-class way to produce a rich continuation prompt before `/compact`; the freeform ad-hoc prompt fires inconsistently and the built-in summary drops the domain model, working discipline, and error log. Add a new **slash command** (not a skill) `command-handoff` at `.claude/commands/command-handoff.md` (its dir and the `~/.claude/commands` symlink are already provisioned — author only the file). Default (no args) emits a dense, session-mined handoff prompt to chat (9-section skeleton: frame → two-tier read-first map → current state → next step → working discipline → error log, +optional orientation/domain-spine/hard-rules), English. A `note`/`ноут` trigger also persists a durable copy **inline** — the command writes the same skeleton to `.ai-factory/notes/<NN>-<slug>.md` itself (semantic slug like aif-note, not a hardcoded `handoff`); it does **not** route through aif-note, which would reshape the handoff into its own note template and lose the skeleton. Also update `CLAUDE.md` (Repository Structure + Upstream Sync, registering `.claude/commands/` as ours and never synced), the `README.md` Setup guide (add the commands symlink), and one line in `ARCHITECTURE.md` registering commands as a hosted artifact type. Spec: `.ai-factory/notes/12-task-command-handoff.md`.

- [x] **roadmap-decompose: always emit two-tier output by invoking aif-note** — The skill writes full specs inline in each ROADMAP bullet (Mode 1 Step 1.3, Mode 2 2.4/2.5, "Roadmap File Format") — no contract/note split, no aif-note call. Getting two-tier today means manually chaining `/roadmap-decompose` + `/aif-note`, which fires inconsistently (`/a /b` parses only `/a`; plus, until now, aif-note's model-invocation ban). Fix: after each Atomicity-Gate-passed task, decompose **invokes the aif-note skill itself** to write the per-task spec note, then emits a roadmap **contract line** — the interface, ~600 chars (range 400–1000), naming key files/types/guards and ending with `` Spec: `.ai-factory/notes/<NN>-<slug>.md`. `` Tell aif-note the content is a task spec but **do not constrain or alter its behavior**. Small surgical change — don't restructure the modes. Supersedes note 09's inline-template mechanism (keeps its concept). Spec: `.ai-factory/notes/13-task-decompose-two-tier-via-aif-note.md`. [26m 26s]

- [x] **aif-docs: state-not-process docs (no-motivation rule + `3d`/`3д` mode)** — Two coupled fixes to `aif-docs`, one theme: docs state factual *state*, not *process*. **A (always-on, both modes):** the skill describes motivation/history ("we changed X to Y because…", "previously…") which the user patches every run — add a Core Principle + Step 4 check that flags and removes it. **B (`3d`/`3д` param):** the skill assumes shipped code (Step 1 "Current State" `:93`, Step 2.1 staleness `:409`, Step 4 accuracy verify `:474`) and **refuses to document not-yet-built features**; a `3d`/`3д` token activates a Document-Driven branch — author docs **as if already shipped** (present-tense target behavior, doc = contract), mostly by **explaining what 3D is** so the agent stops arguing, plus branching those three points (current→target, suppress staleness + accuracy-verify). Keeps all formatting/nav/language rules; drops *current-state*, keeps *no-history*. Small surgical change, no second skill. Marker dropped (softer skill suffices); conformance pointer printed not auto-inserted. Also register `aif-docs` as intentionally diverged in `CLAUDE.md` Upstream Sync (else next sync wipes it). Spec: `.ai-factory/notes/14-task-aif-docs-3d-mode.md`. [11m 1s]

- [x] **milestone-rescue: sidecar lifecycle + non-convergence detection** — Two failure-mode fixes. **A (sidecar lifecycle):** Step 5 always keeps `plans/{slug}.json` and rewrites its `step` even when the plan `.md` is deleted (re-plan) — `step: "planned"` then claims a plan exists when it was just deleted and `planner`/`implementer`/`elapsed` describe a discarded run. Make the sidecar track the plan's lifetime: full reset (plan `.md` deleted) → delete the sidecar too (git-native, elapsed-loss accepted); re-implement (plan + passing plan-reviews kept) → update `step` to `"plan_reviewed"` in place. Carve the exception into the "never delete the sidecar" rule; reconcile the `step` table. **B (non-convergence):** Step 2 treats any no-`REVIEW_PASS` state as a defect and Step 4 always proposes a description change, but a milestone can be done-yet-unrecognized — every review non-blocking, deliverables on disk, reviewer just never emits `REVIEW_PASS` and loops on cosmetic nits. Detect that (no PASS + all rounds non-blocking + deliverables present) and recommend **committing the deliverable** instead of editing the milestone; keep the standard flow when any finding is Blocking/Critical or deliverables are absent. Spec: `.ai-factory/notes/15-task-milestone-rescue-full-reset-deletes-sidecar.md`. [18m 38s]

- [x] **roadmap-decompose: ensure aif-note is loaded once, then write spec notes manually** — Commit `0bfa177` (note 13) made decompose invoke `aif-note` via the Skill tool **once per task**, plus wrong cruft ("pass the spec as a subject so aif-note doesn't blend siblings" — aif-note has no such parameter and handles its own context). Per-task invocation degrades on long phases and the plan-reviewer rejected the plan three times over it. Fix — one simple rule: per task, **ensure `aif-note` was invoked at least once in this chat** (if not, invoke it once; if already, do not re-invoke) so its note-writing instructions are in context, **then write the spec note manually with `Write`**. decompose describes **no** note format of its own (the format comes from aif-note's in-context instructions — add nothing beyond what existed before `0bfa177`) and **does not modify aif-note**. Two-tier shape, contract line, `Spec:` tag, char budget, Atomicity Gate unchanged. Cleanest path: revert `0bfa177`, then re-add two-tier with this rule. Spec: `.ai-factory/notes/16-task-decompose-inline-notes-aif-note-format.md`. [9m 49s]

- [x] **command-handoff: scale handoff detail proportionally to session depth** — `src/commands/command-handoff.md` produces fixed-size output regardless of session scale: skeleton sections default to 1–2 lines each, so a 1M-token architecture session and a trivial bugfix yield handoffs of equal thinness. Fix: add a proportional-depth mandate (enumerate every work-unit individually; never collapse a subsystem to summary bullets), two new first-class sections for substantial sessions (cross-cutting contracts/invariants checklist; per-unit map with watch-points), a concrete named error-log requirement, and a richness self-check gate the agent applies to its own draft before emitting. Guard: mandate is proportional not maximal — short sessions stay short; padding a trivial session is its own failure. Spec: `.ai-factory/notes/18-task-command-handoff-scale-depth.md`. [5m 32s]
