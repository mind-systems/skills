## Test-coverage testability axis — friction, not DI-presence

`roadmap-test-coverage` is stack-parametric everywhere except Layer 5, whose testability-review prompt is an un-abstracted slab of its NestJS origin — a binary DI-presence axis phrased in TS idiom (`process.env`, `(req as any).user`, constructor-vs-module-singleton). A live orchestrator run this session fired `needs-refactor` off it on a Python milestone; the value "inject, don't read from a module singleton" maps badly onto Python, where a module singleton / `os.environ` / `datetime.now()` is trivially `monkeypatch`-able. The misfire still surfaced a real finding (a class self-acquiring its config), so the fix must keep that reachable without the false "untestable" framing. Hard rule for the direction: the silent-failure axis (`test-philosophy`, Layer 3) is a separate concern and stays untouched — this is the orthogonal test-authoring-cost axis; and the verdict token grammar Layer 6/8 consume is a contract.

### Phase 8 — Layer 5's testability axis becomes substitution-friction

- [ ] **8.1 — roadmap-test-coverage: Layer 5 testability review — substitution-friction axis, stack-parametric** — Layer 5's agent prompt (`SKILL.md:158-187`) hardcodes four TS/NestJS checks and a binary DI-presence verdict; on Python it both mistranslates its examples and encodes a language-specific value, so a self-acquired-but-patchable class is flagged for the wrong reason ("not DI") while a naive "patchable? → clean" reframe would lose it. Rewrite the prompt so the axis is **graded on friction-to-substitute-in-a-test**, judged in `$STACK`'s own test idiom: received → `clean`; self-acquired-but-overridable → `needs-refactor` framed as friction ("inject to drop test-setup friction", never "not DI"/"untestable"); unoverridable → strong. Verdict grammar `clean | needs-refactor | area | sentence` byte-identical (Layer 6/7/8 untouched); TS-only checks (`(req as any)`, RxJS teardown) become stack-conditional. Mandatory scope fence: flag coupling only where it taxes test-setup — zero-friction design-smell is code-review's, not this planner's. Guard: don't fold friction into `test-philosophy`'s silent-failure axis; behavior-baseline is a live Layer-5 replay of the misfiring Python case. Spec: `.ai-factory/specs/61-test-coverage-l5-substitution-friction-axis.md`.

### Phase 13 — Doctrine: docs are the ТЗ, they lead code

Sweep `aif-docs` and any doc-update skill for the same lagging framing (`aif-docs` already calls docs a governing spec — the fix removes a self-contradiction, one home in the global file). Independent of the conformance phases — may run first.
