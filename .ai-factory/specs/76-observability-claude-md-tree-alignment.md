## Grove entry checks (run before the pass)

A project family is a **grove**, not a tree: separate git repos, each leaf `CLAUDE.md` committed and published with its own repo, under a coordination root. The four operations below are defined on a tree; the grove adds entry checks — and here, a **nested** grove adds one more:

- **Topology:** confirm the family shape — root + five separate git repos (`observe-write-proxy`, `observe-dart`, `observe-js`, `observe-swift`, `observe-contract`), all confirmed via `git remote -v` inside each. **Plus:** two of those five (`observe-dart`, `observe-swift`) each nest a sixth-and-seventh git repo — a `contract` submodule — one level deeper. These two are excluded from the edit scope (see Current state); confirm on entry that they are still pinned at the same commit as `observe-contract`'s own `HEAD` (a drifted pin would itself be a conflict to escalate, not silently ignore).
- **Hoist premise — the layout guarantee.** Hoist, and deleting leaf-side duplicates of root facts, is licensed only because the root trunk reaches every leaf session *mechanically*: (a) the harness parent-loads CLAUDE.mds up the directory tree across git-repo boundaries, nesting depth included — verify live once; (b) the family layout is **guaranteed** — the root README already mandates checkout structure + freshest-branch switch for five of the six repos, but is missing `observe-contract` (see Current state) — **fix this during the pass**: add `observe-contract` to the clone-command list, the branch-switch list, and the Sub-repositories table. This is a rename-free augmentation (the heading is already correctly scoped, unlike mind's), not a from-scratch add.
- **No upward-edge lines in leaves — ever.** A back-pointer found in a leaf is dedupe input — removed like any duplicate. Symmetrically, the root `CLAUDE.md` never enumerates its consumer projects (n/a here — it lists its own SDKs, which is ownership routing, not upward consumers) — the ownership routing tables (Argument-prefix / keyword tables) stay.
- **Metric re-baseline:** re-measure the six in-scope `CLAUDE.md` sizes on entry and budget "no growth" against that measurement, never against the 402 in Current state.

## Change

One pass over the six in-scope `CLAUDE.md` files (root + five subrepos — **not** the two excluded submodule mirrors), three operations:

- **Dedupe** — a fact stated in two files keeps one home; the second occurrence becomes a pointer to the first (or is deleted when the home is already reachable). Likely candidate, flagged for the pass to confirm: root `CLAUDE.md`'s "## What this is" states "This root repo is the coordination layer" as an aside inside its general intro, and the very next section, "## This repo is the coordinator," restates the same claim as its own heading before elaborating on what the root actually holds (`.ai-factory/`, `.claude/`, `backend/`) — a milder version of the exact intra-file duplicate mind's and tradeoxy's roots each had (there, the whole sentence repeated verbatim in two sections; here it's the same claim in two places, worth collapsing to one home the same way).
- **Hoist** — a meaning repeated across 2+ leaves belongs to the root: move it up, leave pointers (or nothing, where the root is auto-loaded context anyway). The root already holds shared meaning (the cross-project logger-swap table, the OTLP contract summary) and ownership routing; leaf-specific facts stay in leaves. Root's own rules file (`​.ai-factory/rules/base.md`) "Module Structure" section describes the polyrepo layout (`backend/`, `tools/`, the SDK list) that `CLAUDE.md`'s own "Repository structure" and "This repo is the coordinator" sections already cover — a same-repo cross-genre overlap to note, not necessarily fold (see spec 75's equivalent finding and its Guard on cross-genre overlap).
- **Resolve root↔leaf conflicts** — never silently: ground truth from the code/config decides; a conflict ground truth cannot settle is escalated in the report, not invented. **Calibration: conflicts are the expected case** — budget this operation as primary, per 2.1/2.2's own finding rate.

Also enforce while passing: **no skill or command names as routing** in any of the files (the root's own "`observe-logs` skill" mention in "## This repo is the coordinator" is a legitimate description of an existing tool by name, not routing — leave it, the rule targets routing tables pointing at a skill instead of a task-term, which doesn't occur here); present tense, no change-history narration; every `AGENTS.md` in the six in-scope directories is a symlink to its sibling `CLAUDE.md` — content dropped, no salvage, `observe-js`'s 35-line file included (discard unread, same as mind's/tradeoxy's richer AGENTS.md precedent); strip both the `# CLAUDE.md` H1 title and the boilerplate `This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.` intro line, plus the blank lines among/after them, from every in-scope CLAUDE.md — contentless noise, confirmed present as their own standalone lines in all six (root and all five subrepos) — so each file starts directly at its first real `## section`.

**Fourth operation — rules hygiene, converted to the orchestrator's canonical path.** The orchestrator reads a hardcoded `.ai-factory/RULES.md` in every prompt; `.ai-factory/rules/base.md` sits at a path it never opens. Five rules files, all at that orchestrator-invisible path, all sharing the same four-to-five-section shape (`Naming Conventions`, `Module Structure`, `Error Handling`, `Logging`, root only also has `Language`) — pass the counter-default filter (a rule survives iff the executor would do otherwise by default AND code alone cannot teach it, and it carries its why), then the survivors convert to canonical `.ai-factory/RULES.md` (the now-empty `rules/` dirs removed). The filter is followed by a de-duplication pass against CLAUDE.md, governed by a mechanism distinct from the CLAUDE.md tree itself: `RULES.md` is **not** parent-loaded — the orchestrator reads only the target repo's own copy — while `CLAUDE.md` **is** parent-loaded up the directory tree. A rule therefore never hoists to the root's `RULES.md`, which would make it vanish from every leaf's own orchestrator session; each leaf keeps its applicable rules in its own `RULES.md`. The dedup target follows the same asymmetry: a **leaf's** `RULES.md` drops a rule iff the fact is homed in that leaf's own `CLAUDE.md` **or** the root `CLAUDE.md` (both reach the leaf's orchestrator session); the **root's** `RULES.md` dedups against the root `CLAUDE.md` **and** the root `ARCHITECTURE.md` it points to as the contract's home (root `CLAUDE.md` names `ARCHITECTURE.md` as where the shared OTLP contract is defined, "not duplicated here," and the orchestrator reads `ARCHITECTURE.md` at Step 0 of a root session) — a fact reachable through either counts as homed, so the root's own cross-SDK contract facts (uniform public API vocabulary, OTel-convention attribute keys) drop from root `RULES.md` on this basis, and a leaf session, which never reads the root's `ARCHITECTURE.md`, keeps dedupping against leaf + root `CLAUDE.md` only. A cross-SDK counter-default homed in no `CLAUDE.md` (and, for root, no `ARCHITECTURE.md`) stays in each leaf's `RULES.md` individually, flagged in the report, never invented into a `CLAUDE.md` edit. Preview from an entry read: root's "Naming Conventions" reads as **substantive, not noise** at first glance — "per-platform idioms win but the public API surface stays uniform across languages" and "telemetry attribute keys follow OpenTelemetry semantic conventions" are real cross-SDK constraints an implementer could easily violate by defaulting to one platform's idiom — but ground truth settles it the other way: `ARCHITECTURE.md`'s own "Key Principles" already state both ("Uniform public API across platforms... even where language idioms differ"; "Resource attributes identify origin... `project`, `service.name`, `service.instance.id`"), so all three bullets drop and root's `RULES.md` ends up header-only — the correct result for a repo whose genuine facts all live in the doc its own `CLAUDE.md` names as their home. By contrast `observe-js`'s own "Naming Conventions" (`kebab-case.ts` files, `camelCase`/`PascalCase`/`UPPER_SNAKE_CASE`) reads as the standard-TypeScript-convention noise the tradeoxy/mind passes already dropped elsewhere — a likely per-leaf noise target, confirm on entry rather than assume uniformly across all five files (root's is different in kind from the leaves', and resolves to header-only for a different reason: homed elsewhere, not noise). The vestigial `.ai-factory/config.yaml` in all five repos — the orchestrator never reads it — is removed outright. Every kept/dropped rule is listed in the report, each drop citing the `CLAUDE.md` (leaf or root) that homes it.
