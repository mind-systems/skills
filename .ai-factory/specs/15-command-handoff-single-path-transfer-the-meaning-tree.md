# command-handoff: one path — always transfer the session's meaning-tree

**Date:** 2026-07-09
**Source:** conversation context — this session. The skill carries user-facing mode machinery (destination note/chat × register skeleton/narrative = four combos) the user did not intend and does not want; the user does not read handoffs (only the successor agent does), so form is an execution-time choice, not a user knob. The goal stated plainly: a handoff must always be verbose and transfer the full **tree of meanings** the session built — rehydrating the next agent to the understanding of the whole (million-token-scale) session, minus the dross: tool calls and file reads that turned out irrelevant.

## Problem today

`command-handoff.md` exposes two orthogonal user-selectable axes: **destination** (note vs chat, `:13-17`) and **register** (skeleton vs narrative, `:19-23`), selected by user cue or argument (`:23`, `argument-hint` `:8`). That is four combinations the user must (implicitly) pick among — and it is the exact register×destination cross-product whose duplicated invariants made spec 14 loop five review rounds (see this session's audit of milestone 60). The user wants none of that surface: the handoff should always persist to a file and always carry the full perception tree; whether the body comes out as an enumerated grid or as flowing prose is the executing agent's call per session shape, never the user's.

A second, coupled block: the **narrative direct-write path** (`:140-146`) routes around `note` and re-implements `note`'s file-numbering by hand — the drift the milestone-60 code review caught. And the **proportionality guard** (`:55`, `:57`, `:119`, self-check `:123`) is inert in practice (the agent writes to roughly the same depth regardless of context size), so it earns no place as a governing directive.

The one real engine obstacle: `note`'s Rule 2 (`src/skills/note/SKILL.md:105`, reinforced at `:62` "Findings-focus … always apply") is written as an **absolute** — "what was learned, not the process of learning it." The meaning-tree a handoff must carry *is* the process (the causal thread, the branches that were live). Leaving that absolute standing while `command-handoff` demands thread transfer is an internal contradiction — precisely the class of gap that made spec 14 stumble; it must be resolved at the engine, not papered over at the caller.

## The change (two files, one contract)

### 1. `src/skills/note/SKILL.md` — one additive clause

Broaden the **verbosity directive** hook so a caller directive may override **both** the default concision rule (Rule 1, already overridable) **and** the findings-focus/not-process rule (Rule 2, `:105`) — for callers that must preserve the causal thread / process. Strictly additive: unset → today's default (concise, findings-focus) byte-identical; only a caller that sets the directive gets thread preservation. Post-change the always-on set is **Rules 3 (file paths) + 4 (English) only**.

**Sites to reconcile — exhaustive. Rule 2 is asserted always-on in three prose spots; leave any one standing and the hook claims it overridable while the text claims it absolute — the contradiction that looped the plan.** All three are edit targets:
1. `:33` (hook description) — *"all other Important Rules still apply"* → *"all other Important Rules — file paths, English — still apply"* (Rule 2 dropped from the set).
2. `:62`, clause A — *"Findings-focus, file paths, and English always apply to both cases"* → *"File paths and English always apply to both cases."*
3. `:62`, clause B — *"…replaces concision for this run — the other Important Rules are unaffected"* → a set directive replaces **both** Rule 1 and Rule 2 for that run.

Also: the **template hook** (`:32`) is worded as "a section skeleton", but it **tolerates a free-form (non-skeleton) body directive** — the caller passes the body structure to follow verbatim, grid or not; this is what §2's prose shape uses. State it so the implementer does not force a skeleton.

This is the minimal generalization; do not reconceive `note` — its standalone/default behavior, default template, and Steps 1–4 mechanics are untouched.

### 2. `src/commands/command-handoff.md` — collapse to a single path

- **Remove the destination axis** (`:13-17`, and the chat-destination **emit** branch at `:129`): the handoff **always** persists to `.ai-factory/handoffs/`. No chat-only mode. **Sibling site — the frontmatter `description:` block (`:5-7`)** still advertises the mode trigger *"Optionally persist … (pass "note" / "ноут" / "давай ноут" / "с ноутом" as the argument)"*; rewrite its last sentence to state **unconditional** persistence to `.ai-factory/handoffs/` and drop the argument-token list — no mode dispatch may survive anywhere, including the user-facing description (§How-to-verify #1). The minimal paste-back pointer to chat (`:148-164`) **stays** — it is the pointer, not a "chat destination"; its register-conditional bullets `:151-153` are **reworded shape-neutral, not deleted** (see below), so `:151` is not a chat-branch to delete.
- **Remove register as a user-selectable mode** (`:19-23`, the user-cue selection; the `:25` "How Steps 1–2 apply" dispatch; the `:121` per-destination population branches — including its narrative-shape **direct-write** clause, a second copy of the milestone-60 drift separate from `:140-146`): the executing agent still chooses the body **shape** — enumerated grid (wide, many-work-unit session) or flowing prose (a reasoning journey) — inferred from session shape, but **never** a user argument. Strip `argument-hint`'s mode tokens (`:8`) — **post-strip value is empty: `argument-hint: ""`**. Reword the Step-2 opener *"Compose in the register selected above"* (`:51`) → *"Compose in the shape inferred from the session's type."* Keep the 11-section grid and the prose form both available as shapes; delete only the *user-facing dispatch* between them.
- **Reword every surviving heading/bullet that still carries a `register` or `note/chat destination` label — shape-neutral, keep the content (grep-enumerated; these are the sites that looped the re-plan):** `:53` *"**Skeleton register.**"* → *"**Grid shape.**"*; `:117` *"**Narrative register.**"* → *"**Prose shape.**"*; `:131` *"**Note destination — skeleton register:**"* → a single-path delegation heading with **no** destination/register label (under one path both shapes delegate to `note`); the pointer's register-conditional bullets `:151-153` (*"skeleton register: … / narrative register: …"*) → *"grid shape: … / prose shape: …"*. After the change, **`grep -niE "register|note mode|chat mode|destination axis" src/commands/command-handoff.md` returns nothing** outside a deleted block — the tool, not a reading pass, is the completeness check.
- **State the fixed invariant** up front: every handoff is verbose and transfers the full **tree of meanings** the session built — the leaf→branch→trunk perception the successor would otherwise re-derive — so the next agent is rehydrated to the whole session's understanding, **minus the dross**: tool calls and file reads that proved irrelevant are stripped, only the load-bearing tree is carried. (This is signal-compression of a large session; do **not** encode any literal token target — the "rehydrate to ~200k" the user spoke is a metaphor for that success, not a number.)
- **Delegate numbering/placement to `note`** (loaded once via `loads: note`), for both shapes: **delete the direct-`Write` path and its hand-rolled `<NN>` numbering (`:140-146`)** — `note` owns numbering, so the milestone-60 drift cannot recur. `command-handoff` supplies `note` the hooks: **destination** `.ai-factory/handoffs/`; **template** = the chosen shape — the blank 11-section skeleton for the grid, or, for the prose shape, a **free-form body directive** through the same template hook (`note` tolerates a non-skeleton directive, §1 — do not force a skeleton); **verbosity/fidelity directive** = "verbose; carry the full meaning-tree and its causal thread; strip irrelevant tool-calls and dead-end reads" (exercises note's new Rule-2 override); **slug** semantic, never the literal word `handoff`. `note`'s own Step-4 report stays **not surfaced** (`:138`) on the single path — only the minimal pointer is emitted.
- **Drop the proportionality guard** as a governing directive — **all its sites**: the proportionality prose at `:55` and `:119`, and the proportionality clauses of the self-check gate (`:123`). Replace the gate with a **tree-completeness** check: *could a fresh agent, from this handoff alone, hold the same perception tree — leaf→branch→trunk — the session ended with, without the irrelevant exploration?* **Sibling reference — `:155`** (inside the kept paste-back pointer) quotes *"exempt from the proportionality gate"*; re-anchor it, **and** the exemption clause inside the `:123` self-check, to the **tree-completeness** gate — the minimal chat pointer is exempt from carrying the full tree (that lives in the persisted file). Leave **no orphaned quotation** of the removed proportionality text.
- **Preserve spec 14's boundary-neutral framing** — no compact claim survives, no boundary taxonomy is introduced (already true post-milestone-60; do not regress).
