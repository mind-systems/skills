**1. (Low — report inaccuracy, Step 8) Default-pair branch omits flat `test-runs/` for a `ROADMAP_TESTS.md` prune.**
`src/skills/roadmap-prune/SKILL.md:349-352`. Step 8 now enumerates: *"the flat three dirs for a default-pair prune, or the pruned stem's `plans/<stem>/`, `plan-reviews/<stem>/`, `reviews/<stem>/` (plus `test-runs/<stem>/` in tests mode) for a named prune."* The `(plus test-runs/ in tests mode)` qualifier is attached **only** to the named branch. So a default test-mode prune sweeps four dirs (`plans/`, `plan-reviews/`, `reviews/`, `test-runs/`) yet Step 8 tells the summary to list only "the flat three dirs" — under-reporting `test-runs/`.
Failure scenario: prune `ROADMAP_TESTS.md` in a solo repo → `test-runs/` is deleted but the Step-8 summary omits it. Report-only, never gates, so harmless to the prune itself — but it is a real inaccuracy the change introduced (the pre-change wording "List the dirs swept in Step 5" was generically accurate; the new specific enumeration lost the default test-runs case). Fix: mirror the parenthetical onto the default branch too — e.g. "the flat three dirs (plus flat `test-runs/` in tests mode) for a default-pair prune".
